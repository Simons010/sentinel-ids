## 2024-05-24 - Avoiding N+1 queries in 24-hour exact rolling window
**Learning:** When generating time-series data for an exact "rolling" 24-hour window, iterating over 24 individual hour buckets and querying the database for each bucket using `.count()` creates severe N+1 problems (e.g., 72 sequential queries per API request). Using database `TruncHour` aggregation is also problematic because it aggregates based on absolute calendar hours, which breaks exact rolling 24-hour window boundaries.
**Action:** Always fetch minimal sparse columns as a list using `.values()` and perform the grouping and summation in pure Python memory iteration. Additionally, combine multiple sequential `.count()` queries on the same queryset into a single database `.aggregate()` using `Count('id', filter=Q(...))` to significantly reduce roundtrips.

## 2024-04-24 - N+1 Queries in Exact 24-hour Rolling Window Approximations
**Learning:** Found an architectural anti-pattern in how the Django backend handles 24-hour rolling windows (`DashboardStatsView`, `AnalyticsView`). Because the interval isn't absolute calendar hours (where we could use `TruncHour`), developers looped `range(24)` and did 3 `Count()` queries per iteration to bucket by exactly `now() - timedelta(...)`. This caused 72 sequential queries per API call on views that are hit frequently.
**Action:** When aggregating exact rolling windows, avoid `.count()` in a loop. Fetch minimal sparse columns using `.values("timestamp", "is_suspicious")` as a list, and perform the summation mathematically in pure Python memory. This cuts DB calls from N*3 to exactly 2 queries.

## 2024-05-14 - Replace Multiple Count Queries with Aggregate
**Learning:** Found a common performance anti-pattern in Django views where multiple `.count()` queries and Python-side iterations are used to compute stats that can all be collected in a single database round trip. Specifically, the `AnalyticsView.get` was making 6 separate `.count()` calls and fetching all model score values (`values_list(..., flat=True)`) into Python memory just to compute bucket distributions.
**Action:** Combined multiple `.count()` calls and the score distribution calculations into a single database hit using Django's `.aggregate()` function along with `django.db.models.Q` to filter the conditional counts directly in the database. This drastically reduces database round trips and memory consumption when calculating statistics.

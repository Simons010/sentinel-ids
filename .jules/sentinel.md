## 2024-04-18 - [Missing Global Authentication Fallback in DRF]
**Vulnerability:** The `REST_FRAMEWORK` settings in `backend/backend/settings.py` lacks a `DEFAULT_PERMISSION_CLASSES` fallback. As a result, all DRF views default to `AllowAny`, leaving critical dashboard, alert, report, and settings API endpoints fully exposed to unauthenticated users.
**Learning:** In this specific codebase configuration, user-facing DRF APIViews must explicitly declare `permission_classes = [IsAuthenticated]` (or an equivalent) because there is no global protection layer. Applying these indiscriminately can break automated ingestion endpoints (`LogIngestView`, `LogUploadView`) which might use different authentication models like API keys. Furthermore, class-level attributes should be added after the class docstring to avoid breaking string formatting or throwing errors.
**Prevention:** Either define a secure fallback in `DEFAULT_PERMISSION_CLASSES` across `REST_FRAMEWORK` and conditionally bypass it, or stringently review all new DRF view implementations to ensure `permission_classes` are not omitted.

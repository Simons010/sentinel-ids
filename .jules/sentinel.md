## 2024-05-18 - Missing Authentication on Sensitive API Endpoints
**Vulnerability:** Django REST Framework API endpoints handling sensitive data (Dashboard, Reports, Analytics, Settings, Alerts) lacked any authentication checks (`IsAuthenticated`), leaving them publicly accessible. The `REST_FRAMEWORK` configuration in settings lacked a fallback `DEFAULT_PERMISSION_CLASSES`.
**Learning:** We must not rely on implicit or default authentication in Django Rest Framework, especially when a global `DEFAULT_PERMISSION_CLASSES` is omitted in the backend settings. Every view must explicitly declare its access requirements. Automated integration endpoints like `LogIngestView` make applying generic global session-based auth difficult, leading to unprotected user-facing views if omitted.
**Prevention:** Always define explicit `permission_classes = [IsAuthenticated]` (or a custom API Key permission for integrations) on every `APIView`. We should consider adding a custom default permission class and explicitly overriding it with `AllowAny` on ingestion endpoints if needed, rather than defaulting to open access.

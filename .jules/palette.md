## 2024-05-18 - TopBar Keyboard Accessibility & ARIA Overhaul
**Learning:** Hardcoding dynamic values (like unread counts) into static `aria-label` attributes causes screen readers to go out of sync with visual state. The correct pattern is to inject visually hidden (`sr-only`) text alongside the visual elements and apply `aria-hidden="true"` to the parent or purely visual components. Also, icon-only buttons like the Sidebar Toggle and Search need explicit `aria-label`s, and `focus-visible` states are critical for keyboard navigation, particularly in complex global navigation components like the TopBar.
**Action:** Always prefer `sr-only` text for dynamic states over `aria-label`, and routinely audit icon buttons for `aria-label` and `focus-visible` coverage in Sentinel-IDS components.

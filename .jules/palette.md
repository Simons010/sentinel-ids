## 2024-05-24 - Accessibility improvements for TopBar
**Learning:** Found that Sentinel-IDS's top bar component had several icon-only buttons (like notifications bell and sidebar toggle) without ARIA labels, rendering them opaque to screen readers. Also observed that purely decorative icons inside buttons and visually hidden dynamic status indicators (like the green "LIVE" badge) were not hidden from assistive tech. Furthermore, keyboard navigation was not clearly outlined using Tailwind's `focus-visible` states.
**Action:** Always ensure `aria-label` is used for icon-only buttons, `aria-hidden="true"` is attached to purely decorative inner elements (so screen readers don't read the icon name), and apply `focus-visible:ring-2 focus-visible:outline-none` for distinct keyboard focus states across all interactive elements in the application header.
